A modern C++ implementation of the Open Sound Control (OSC) protocol based on the OSC 1.\+0 and 1.\+1 specifications. This library provides a robust, type-\/safe, and cross-\/platform interface for OSC communication.\hypertarget{md__r_e_a_d_m_e_autotoc_md13}{}\doxysection{\texorpdfstring{Features}{Features}}\label{md__r_e_a_d_m_e_autotoc_md13}

\begin{DoxyItemize}
\item Support for all OSC data types including int32, int64, float, double, string, blob, and more {\itshape (Implemented)}
\item Bundle support with immediate or timestamped message delivery {\itshape (Implemented)}
\item Multiple transport protocols (UDP, TCP, Unix sockets) {\itshape (Foundation Laid, TCP Framing Missing)}
\item Cross-\/platform compatibility (Windows, mac\+OS, Linux) {\itshape (Foundation Laid)}
\item Modern C++ design with RAII, strong typing, and exception safety {\itshape (Implemented)}
\item Thread-\/safe server implementation ({\ttfamily Server\+Thread})
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md14}{}\doxysection{\texorpdfstring{Requirements}{Requirements}}\label{md__r_e_a_d_m_e_autotoc_md14}

This implementation aims to adhere to both OSC 1.\+0 and 1.\+1 specifications, supporting\+:


\begin{DoxyItemize}
\item Standard OSC address pattern syntax and matching
\item All core OSC data types
\item OSC bundles with time tags
\item OSC 1.\+1 extensions (arrays, etc.)
\end{DoxyItemize}

\begin{DoxyEnumerate}
\item {\bfseries{CMake Not Found}}\+: Ensure CMake is installed and in your PATH
\item {\bfseries{Compilation Errors}}\+:
\begin{DoxyItemize}
\item Check your compiler supports C++17
\item On Windows, ensure Visual Studio and Windows SDK are properly installed
\item On Linux, make sure you have the pthread development libraries installed
\end{DoxyItemize}
\item {\bfseries{Socket Initialization Failures}}\+:
\begin{DoxyItemize}
\item On Windows, check that Win\+Sock libraries are available
\item On Linux/mac\+OS, check your firewall settings if socket binding fails
\end{DoxyItemize}
